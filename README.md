# Medical AI-Generated Text Detection

Research project on detecting AI-generated text in the medical domain, developed for the HSE University Master's in Applied AI seminar (2026).

The project compares lexical, statistical, recurrent, pretrained-embedding, and transformer classifiers. Its central finding is that excellent in-domain performance can be misleading: models trained only on HC3 Medicine learn dataset-specific style artifacts and generalize poorly to independent medical writing.

> This is a research prototype, not a clinical, academic-integrity, or moderation product. Model scores must not be treated as proof that a text was generated by AI.

## Research Question

Can a detector distinguish human-written from AI-generated medical text across sources, rather than merely learning artifacts from one benchmark?

## Key Results

| Model | HC3 test accuracy | External MedlinePlus accuracy |
|---|---:|---:|
| TF-IDF + Logistic Regression | 99.74% | 42.74% |
| BiLSTM | 96.63% | 40.68% |
| BiGRU | 95.85% | 52.24% |
| TinyBERT | 97.67% | 55.73% |
| GloVe + BiLSTM | 97.67% | 46.42% |
| Statistical features | 93.04% | 35.22% |
| Cross-source TF-IDF, default threshold | - | 94.66% |
| Cross-source TF-IDF, tuned threshold | - | 94.07% |

The first HC3 baseline reached 94.56% accuracy using only the first ten words. Human answers frequently begin with forum-specific greetings, while ChatGPT answers use assistant-style explanatory openings. External evaluation confirmed that these artifacts do not transfer reliably.

Adding source diversity to training improved robustness more than changing model architecture. The updated cross-source model achieved **95.87% ROC-AUC**; its default threshold produced 100% human recall but only 43.75% AI recall, while threshold calibration improved the balance. See [`reports/`](reports/) for complete metrics, confusion matrices, ablations, and conclusions.

## Data

- **HC3 Medicine**: paired human and early-ChatGPT medical answers.
- **MedlinePlus Human**: 1,016 English patient-education topics.
- **MedlinePlus Generated**: 100 AI-generated topic explanations with generator provenance.
- **Cross-source splits**: HC3 and MedlinePlus examples grouped to reduce source leakage.

Datasets are not committed. See [`data/README.md`](data/README.md) for sources, layout, and preparation commands.

## Setup

Python 3.13 is used by the current environment.

```bash
git clone https://github.com/NabilM5/AI_Detection.git
cd AI_Detection
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Run all scripts from the repository root because paths are repository-relative.

## Reproduce the Core Pipeline

### 1. Prepare HC3 Medicine

```bash
python scripts/download_med_data.py
python src/data/prepare_hc3_jsonl.py
python src/data/make_splits.py
```

### 2. Prepare MedlinePlus

```bash
python src/data/download_medlineplus.py
python src/data/generate_medlineplus_ai.py
python src/data/make_medlineplus_splits.py
python src/data/make_cross_source_splits.py
```

AI generation is optional and requires provider credentials in `.env`; start from `.env.example`.

### 3. Train Models

```bash
python src/baselines/train_tfidf_logreg.py
python src/baselines/train_lstm.py
python src/baselines/train_gru.py
python src/baselines/train_tinybert.py
python src/baselines/train_pretrained_embedding_lstm.py
python src/baselines/train_tfidf_cross_source.py
python src/statistics/train_statistical_classifier.py
```

### 4. Evaluate External Generalization

```bash
python src/eval/evaluate_medlineplus_combined.py
python src/eval/evaluate_lstm_medlineplus_combined.py
python src/eval/evaluate_gru_medlineplus_combined.py
python src/eval/evaluate_tinybert_medlineplus_combined.py
python src/eval/evaluate_pretrained_embedding_lstm_medlineplus_combined.py
python src/statistics/evaluate_statistical_classifier_external.py
```

## LLM Provider Configuration

Copy the example and add one provider's credentials:

```bash
cp .env.example .env
```

Generic configuration:

```env
LLM_PROVIDER=openai_compatible
LLM_API_KEY=your-real-api-key
LLM_MODEL=openrouter/free
LLM_BASE_URL=https://openrouter.ai/api/v1
```

The provider-neutral client supports OpenAI-compatible endpoints, Anthropic, and Google AI Studio. Never commit `.env`; it is excluded by `.gitignore`.

## Limitations

- HC3 contains strong source and opening-style artifacts.
- The external benchmark is class-imbalanced and includes generated data from several providers.
- Generator drift, paraphrasing, editing, and mixed-authorship text remain difficult.
- Probability calibration does not establish authorship certainty.
- Medical-domain performance does not imply safety for clinical decisions.
